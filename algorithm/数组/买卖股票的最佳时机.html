源：买卖股票的最佳时机

给定一个数组 prices ，它的第 i 个元素 prices[i] 表示一支给定股票第 i 天的价格。

你只能选择 某一天 买入这只股票，并选择在 未来的某一个不同的日子 卖出该股票。设计一个算法来计算你所能获取的最大利润。

返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。

暴力解法版本 #

通过两层循环找到最大利润：

js
function maxProfit(prices) {
    let maxProfit = 0;

    for (let i = 0; i < prices.length; i++) {
        const minPrice = prices[i];
        for (let j = i + 1; j < prices.length; j++) {
            maxProfit = Math.max(maxProfit, prices[j] - minPrice);
        }
    }

    return maxProfit;
}

单次循环优化版本 #

针对这个问题，“买卖股票的最佳时机”，你需要在给定的价格数组中找到一个买入点和一个卖出点，使得卖出点的价格减去买入点的价格最大化。这个问题可以通过一次遍历数组来解决，使用动态规划的思想，但实际上可以简化为记录到当前位置为止的最低价格，并计算当前价格与这个最低价格之间的差值作为潜在利润，不断更新最大利润。

以下是JavaScript实现：

javascript
function maxProfit(prices) {
    let minPrice = prices[0]; // 初始化最低价格为第一天的价格
    let maxProfit = 0; // 初始化最大利润为0

    for (let i = 1; i < prices.length; i++) {
        // 更新最低价格
        if (prices[i] < minPrice) {
            minPrice = prices[i];
        }
        // 计算当前价格与最低价格之间的利润，并更新最大利润
        else if (prices[i] - minPrice > maxProfit) {
            maxProfit = prices[i] - minPrice;
        }
    }

    return maxProfit;
}

// 示例
const prices = [7, 1, 5, 3, 6, 4];
console.log(maxProfit(prices)); // 输出: 5
// 解释: 在第2天（股票价格=1）的时候买入，在第5天（股票价格=6）的时候卖出, 这笔交易所能获取利润=6-1=5 。
// 注意：你不能在第1天买入第2天卖出，因为这样就不是在未来不同的日子卖出了。

这个算法的时间复杂度是O(n)，因为我们只需要遍历数组一次。空间复杂度是O(1)，因为我们只使用了常量级别的额外空间来存储minPrice和maxProfit。这个算法非常高效，因为它避免了不必要的比较和存储，只保留了到当前位置为止的关键信息。

实现过程 #

代码的主要目的是在给定的股票价格数组 prices 中找到一种买卖策略，使得卖出股票时获得的利润最大化。具体来说，代码通过一次遍历数组来找到最佳的买入和卖出点。

  1. 初始化：

    • minPrice 被初始化为 prices[0]，即假设第一天就是买入股票的日子（实际上这只是为了开始遍历而做的假设，真正的买入点会在遍历过程中确定）。
    • maxProfit 被初始化为 0，表示到目前为止还没有任何利润。

  2. 遍历数组：

    • 从数组的第二个元素（索引为 1）开始遍历，因为我们需要至少两天的价格来比较并可能进行买卖。
    • 在遍历过程中，对于每一天的价格 prices[i]：
      • 更新最低价格：如果当前价格 prices[i] 比迄今为止记录的最低价格 minPrice 还低，则更新 minPrice 为 prices[i]。这是因为我们可能找到了一个新的、更便宜的买入点。
      • 计算并更新最大利润：否则（即当前价格不低于最低价格），我们计算如果今天卖出股票能获得的利润（prices[i] - minPrice），并将其与迄今为止记录的最大利润 maxProfit 进行比较。如果这个利润大于 maxProfit，则更新 maxProfit。

  3. 返回结果：

    • 遍历完成后，maxProfit 中存储的就是可以获得的最大利润，将其返回作为结果。

思路 #

这个算法的思路基于贪心算法的思想，但它并不直接做出“贪心”的选择（即每一步都选择当前看来最好的），而是通过维护一个“到目前为止的最低价格”来间接实现最优解。在遍历过程中，我们不断地寻找可能的买入点（即迄今为止的最低价格），并计算如果在这个买入点买入并在当前点卖出能获得的利润。由于我们只需要找到一次买入和一次卖出的最佳时机，因此这种方法是有效的。

值得注意的是，虽然算法在遍历过程中似乎做出了多次“买入”的假设（即每次遇到更低的价格就更新 minPrice），但实际上这只是一种计算潜在利润的方式，真正的买入点只有一个（即遍历过程中记录的最低价格点）。同样地，卖出点也是唯一的，它对应于遍历过程中使得 prices[i] - minPrice 最大的那个点。因此，这个算法实际上是在模拟了一种只进行一次买入和一次卖出的交易策略，并找出了这种策略下的最大利润。

使用动态规划的思想 #

虽然“买卖股票的最佳时机”问题通常不直接需要动态规划（DP）来解决（因为可以通过一次遍历找到最优解），但我们可以将其转化为一个动态规划问题来理解。下面是一个使用动态规划思想的解法，尽管它的效率与一次遍历的解法相同，但展示了动态规划的应用。

在这个动态规划解法中，我们定义一个数组 dp，其中 dp[i] 表示在第 i 天卖出股票时能获得的最大利润。但是，由于我们实际上只关心到某一天为止能获得的最大利润，而不是具体在哪一天卖出，我们可以稍微简化这个问题，只使用两个变量来跟踪到目前为止的最低价格和最大利润。

不过，为了展示动态规划的思想，我们可以假设 dp[i] 是在考虑前 i 天价格的情况下，我们能获得的最大利润。然后，我们可以写出状态转移方程：

js
dp[i] = max(dp[i-1], prices[i] - minPriceSoFar)

其中 minPriceSoFar 是从第 0 天到第 i-1 天中的最低价格。但是，我们不需要真的存储整个 dp 数组，因为我们可以只用一个变量来跟踪 dp[i-1] 的值（即到目前为止的最大利润），并用另一个变量来跟踪 minPriceSoFar。

下面是使用这种“动态规划”思想的代码实现（但实际上是空间优化的版本）：

javascript
function maxProfit(prices) {
    if (prices.length <= 1) return 0; // 如果数组长度小于等于1，则无法买卖股票，返回0

    let maxProfit = 0; // 到目前为止的最大利润
    let minPriceSoFar = prices[0]; // 到目前为止的最低价格

    for (let i = 1; i < prices.length; i++) {
        // 更新到目前为止的最低价格
        if (prices[i] < minPriceSoFar) {
            minPriceSoFar = prices[i];
        }
        // 根据状态转移方程更新最大利润
        else if (prices[i] - minPriceSoFar > maxProfit) {
            maxProfit = prices[i] - minPriceSoFar;
        }
        // 注意：这里我们没有显式地更新dp[i]，因为dp[i]的值就是maxProfit
    }

    return maxProfit;
}

// 示例
const prices = [7, 1, 5, 3, 6, 4];
console.log(maxProfit(prices)); // 输出: 5

虽然这个解法在表面上看起来像是动态规划，但实际上它只使用了动态规划的思想（即通过子问题的解来构建原问题的解）和状态转移方程，而没有显式地存储所有子问题的解（即没有使用DP表）。这种优化后的解法在空间复杂度上更优，因为它只使用了常量级别的额外空间。

纯动态规划版的代码 #

以下是一个使用动态规划思想的代码示例，但请注意，这个解法在实际应用中并不比一次遍历的解法更高效，因为它仍然只使用了常量级别的额外空间（除了输入数组本身），并且时间复杂度也是O(n)。不过，它展示了动态规划的基本思想。

javascript
function maxProfitDP(prices) {
    if (prices.length <= 1) return 0;

    // 创建一个数组来存储每一天卖出股票时的最大利润（但实际上我们不会用到整个数组）
    let dp = new Array(prices.length).fill(0);

    // 初始化第一天之前的最低价格为正无穷大（实际上我们从第二天开始考虑）
    // 和到目前为止的最大利润为0
    let minPriceSoFar = Infinity;
    let maxProfit = 0;

    // 动态规划过程（但只使用了dp的思想，没有真正填充dp数组）
    for (let i = 0; i < prices.length; i++) {
        // 更新到目前为止的最低价格
        if (prices[i] < minPriceSoFar) {
            minPriceSoFar = prices[i];
        }
        // 如果我们到了最后一天，或者当前价格减去最低价格的利润大于之前的最大利润
        // 则更新最大利润（注意：这里我们并没有显式地填充dp[i]）
        if (i === prices.length - 1 || prices[i] - minPriceSoFar > maxProfit) {
            maxProfit = Math.max(maxProfit, prices[i] - minPriceSoFar);
            // 如果要填充dp数组，这里应该是 dp[i] = maxProfit，但我们实际上不需要它
        }
    }

    // 返回最大利润
    return maxProfit;
}

// 示例
const prices = [7, 1, 5, 3, 6, 4];
console.log(maxProfitDP(prices)); // 输出: 5

// 注意：上面的代码并没有真正利用dp数组来存储中间结果，
// 它只是展示了动态规划的思想，并在实际操作中进行了优化。
